3D implementation rule
Important: 3D is an enhancement, not the product. The full app must work without heavy 3D. Use CSS 3D + Framer Motion first. Use Three.js only for homepage/template showcase. Mobile gets reduced animation to protect speed and battery.
Technology choices
Framer MotionPage transitions, envelope reveal, card hover, dashboard animations, RSVP success states.
CSS 3D transformsCard flip, tilt, template hover, envelope open. Fast and cheaper than heavy 3D rendering.
Three.js / R3FOnly for homepage hero and optional advanced template carousel. Lazy-loaded after first paint.
Canvas particlesSmall event-based particles. Disabled/reduced for low-end devices and prefers-reduced-motion.
Complete user journey — from landing to paid invite
Step 1 · Public
User lands on homepage
User sees 3D hero, sample invite animation, event categories, templates preview, pricing and FAQ. No login required.
Step 2 · Public
Browse categories and templates
User selects category such as Wedding, Birthday, Housewarming, Baby, Festival or Corporate. Each category has visible templates.
Step 3 · Public preview
Open a demo invite
User experiences envelope reveal, card preview, Telugu/English/Hindi language switch, RSVP demo and watermark.
Step 4 · Try editor
Customize before signup
User can change names, date, location, message and optional image temporarily. Watermark remains. Save/send/export locked.
Step 5 · Conversion point
Login required only when saving
When user clicks Save, Send, Export or Remove Watermark, app asks for signup/login. This avoids early bounce.
Step 6 · Dashboard
Create event and manage guests
Logged-in user sees dashboard, event list, invite editor, guest manager, share link, QR code and RSVP tracking.
Step 7 · Monetization
Upgrade when value is clear
Free plan keeps watermark and limits guests/templates. Paid plan removes watermark and unlocks premium features.

Security and content protection
Reality: Browsers cannot fully stop screenshots. Users can use another phone, OS screenshot, screen recording or browser tools. The correct business solution is watermarking, premium unlock, access control and abuse prevention.
S1
Watermark free previews — non-paid and demo previews show “Created with Celebr8” watermark across preview and public demo.
S2
Remove watermark after payment — only paid plan/event removes watermark on public invite and downloadable/printable assets.
S3
Public invite token — invite URLs use random, non-guessable codes, not sequential event IDs.
S4
Row-level security — users can only read/update their own events, guests and payments.
S5
Rate limiting — protect RSVP submit, AI FAQ, auth and CSV upload from abuse.
S6
Input sanitization — sanitize invite message, names, venue and custom text to prevent XSS.
S7
Image upload rules — file size limit, allowed formats, virus scanning/provider checks, private storage if needed.
S8
Privacy pages — Terms, Privacy Policy, Refund Policy, Cookie notice and account deletion path before paid launch.
S9
Admin controls — admin can disable abusive templates/events, view reports, manage template status.
S10
Analytics privacy — track opens/RSVPs without exposing guest personal details publicly.

Golden rules
R1
Do not build 680 templates first. Launch with 50 high-quality templates and validate demand.
R2
Do not force login too early. Let users preview and edit first; ask login at save/send/export.
R3
Do not automate WhatsApp in MVP. Use manual deeplink first to avoid cost and approval delay.
R4
Database before UI. No fake dashboard that cannot persist data.
R5
Mobile first. Most users will open invites on phones.
R6
Performance is product. Premium animation must not make app slow.
