Final MVP product contract
Build goal: Create a live, responsive, production-quality invitation web app where users can browse 50 premium templates, preview/edit an invite before login, sign up to save, share via public link/WhatsApp/QR, collect RSVP responses, track guests in dashboard, and pay to remove watermark/unlock premium limits.

Non-negotiable product rules
R1
Do not build as a static prototype. Every dashboard feature must connect to Supabase and persist data.
R2
Do not require login to browse or try. Login appears only when saving, sending, exporting, or removing watermark.
R3
Do not implement paid WhatsApp API in MVP. Use WhatsApp deeplink/manual share first to keep cost low.
R4
Do not promise screenshot blocking. Use watermark overlay, paid unlock, and clear product rules instead.
R5
Mobile-first required. Every screen must work cleanly at 390px, tablet, and desktop.

Public

User sees hero, examples, categories and Create Invite CTA. No login required.
Public

From /templates or homepage featured templates. User can filter by Wedding, Birthday, Housewarming, Baby, Festival, Corporate.

3. User customizes sample
Fields save only in browser local/session state. Show watermark and a sticky CTA: “Create account to save this invite.”
Auth gate

Prompt login/signup. After auth, convert temporary draft into a saved event.

Production database schema — Supabase/Postgres
Instruction for Claude Code: create this as a Supabase migration. Enable RLS on all user-owned tables. Add indexes before testing dashboard performance.
-- Enable extensions
create extension if not exists "uuid-ossp";
create extension if not exists pgcrypto;

-- Enums
create type event_status as enum ('draft','published','archived');
create type rsvp_status as enum ('pending','accepted','maybe','declined');
create type plan_key as enum ('free','starter','premium','family','corporate');
create type template_category as enum ('wedding','birthday','housewarming','baby','festival','corporate');

-- Profiles linked to Supabase auth.users
create table public.profiles (
  id uuid primary key references auth.users(id) on delete cascade,
  full_name text,
  email text not null,
  phone text,
  country text default 'US',
  default_language text default 'en' check (default_language in ('en','te','hi')),
  timezone text default 'America/New_York',
  avatar_url text,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

create table public.templates (
  id uuid primary key default gen_random_uuid(),
  slug text unique not null,
  name text not null,
  category template_category not null,
  base_layout text not null,
  is_premium boolean default false,
  supports_photo boolean default false,
  supported_languages text[] default array['en','te','hi'],
  thumbnail_url text,
  preview_image_url text,
  config jsonb not null default '{}'::jsonb,
  default_copy jsonb not null default '{}'::jsonb,
  status text default 'active' check (status in ('active','hidden','draft')),
  sort_order int default 0,
  created_at timestamptz default now()
);

create table public.plans (
  key plan_key primary key,
  name text not null,
  price_usd numeric(10,2) default 0,
  max_events int,
  max_guests_per_event int,
  premium_templates boolean default false,
  remove_watermark boolean default false,
  csv_import boolean default false,
  rsvp_dashboard boolean default false,
  stripe_price_id text
);

create table public.subscriptions (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references public.profiles(id) on delete cascade,
  plan plan_key not null default 'free',
  status text not null default 'active',
  stripe_customer_id text,
  stripe_subscription_id text,
  current_period_end timestamptz,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

create table public.events (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references public.profiles(id) on delete cascade,
  template_id uuid references public.templates(id),
  title text not null,
  category template_category not null,
  status event_status default 'draft',
  invite_code text unique not null default encode(gen_random_bytes(8), 'hex'),
  default_language text default 'en' check (default_language in ('en','te','hi')),
  event_date date,
  event_time time,
  timezone text default 'America/New_York',
  venue_name text,
  venue_address text,
  map_url text,
  rsvp_deadline timestamptz,
  is_watermark_removed boolean default false,
  published_at timestamptz,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

create table public.event_content (
  id uuid primary key default gen_random_uuid(),
  event_id uuid not null references public.events(id) on delete cascade,
  language text not null check (language in ('en','te','hi')),
  host_names text,
  headline text,
  message text,
  custom_fields jsonb default '{}'::jsonb,
  photo_url text,
  unique(event_id, language)
);

create table public.guests (
  id uuid primary key default gen_random_uuid(),
  event_id uuid not null references public.events(id) on delete cascade,
  name text not null,
  email text,
  phone text,
  guest_group text default 'family',
  preferred_language text default 'en' check (preferred_language in ('en','te','hi')),
  invite_code text unique not null default encode(gen_random_bytes(8), 'hex'),
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

create table public.rsvps (
  id uuid primary key default gen_random_uuid(),
  event_id uuid not null references public.events(id) on delete cascade,
  guest_id uuid references public.guests(id) on delete set null,
  guest_name text not null,
  status rsvp_status not null,
  plus_ones int default 0 check (plus_ones >= 0),
  message text,
  dietary_preference text,
  submitted_at timestamptz default now(),
  unique(event_id, guest_id)
);

create table public.invite_opens (
  id uuid primary key default gen_random_uuid(),
  event_id uuid not null references public.events(id) on delete cascade,
  guest_id uuid references public.guests(id) on delete set null,
  invite_code text,
  user_agent text,
  ip_hash text,
  opened_at timestamptz default now()
);

create table public.payments (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references public.profiles(id) on delete cascade,
  event_id uuid references public.events(id) on delete set null,
  plan plan_key,
  amount numeric(10,2),
  currency text default 'usd',
  stripe_checkout_session_id text,
  stripe_payment_intent_id text,
  status text default 'pending',
  created_at timestamptz default now()
);

create table public.audit_logs (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references public.profiles(id) on delete set null,
  action text not null,
  entity_type text,
  entity_id uuid,
  metadata jsonb default '{}'::jsonb,
  created_at timestamptz default now()
);

Billing logic
B1
Free preview: user can test editor, but saving/publishing requires account. Public demo includes watermark.
B2
Starter/Premium event purchase: payment can unlock one specific event. Set events.is_watermark_removed = true after successful webhook.
B3
Family monthly: active subscription unlocks plan limits while subscription status is active.
B4
Premium templates: selecting a premium template is allowed for preview but saving/publishing requires Premium/Family or event upgrade.
B5
Watermark removal: remove only after confirmed Stripe webhook, not immediately after checkout redirect.

Production security requirements
Screenshot reality: browsers cannot truly block screenshots. Implement visible watermark, paid unlock, no high-resolution export until paid, and legal terms instead.
S1
RLS required: profiles, events, event_content, guests, payments, subscriptions and RSVP owner dashboard data must use row-level security.
S2
Public invite safety: /invite/[code] must expose only event data required for guests. Do not expose owner email, billing, internal IDs.
S3
Input validation: validate all API inputs with Zod or equivalent. Reject oversized strings and unsupported languages/categories.
S4
XSS protection: sanitize user invite message and render text safely, never dangerouslySetInnerHTML for user content.
S5
Rate limit: RSVP submissions, AI FAQ, auth-sensitive endpoints and CSV import.
S6
File upload: only jpg/png/webp, max 5MB for MVP, store with private owner path, generate public signed URLs only when needed.
S7
Webhook verification: Stripe webhook must verify signature using STRIPE_WEBHOOK_SECRET.
S8
Service role key: never use service role key in client components. Server route only.
S9
Audit logs: log event creation, payment unlock, guest import, RSVP export, delete event.
S10
Legal pages: Terms, Privacy, Refund Policy and Contact must exist before payment launch.
